# 0.8.0 Release Notes Draft

## AssegaiPHP Core 0.8.0

`0.8.0` is the OpenSwoole runtime foundations release.

This is a runtime architecture milestone. It gives AssegaiPHP a real alternate runtime path instead of treating the PHP built-in server model as the only way the framework can run.

The headline here is not just “OpenSwoole support exists.”

The more important change is that the framework now has the runtime architecture needed to support a long-lived worker model safely.

## Highlights

### Experimental OpenSwoole runtime support

AssegaiPHP can now be served through an experimental OpenSwoole runtime path for validation and early adoption.

That includes:

- runtime selection through the CLI
- project-level OpenSwoole server configuration
- request/response bridging for the alternate runtime
- application bootstrap and shutdown behavior that fits long-lived workers better

### Safer request isolation in long-lived workers

A lot of work in this release went into correctness.

The framework now does a better job of separating:

- application-scoped state
- request-scoped state
- runtime-specific response emission

That makes sequential requests inside one worker much safer than they would be under a short-lived-request-only design.

### Better lifecycle support

`0.8.0` also adds clearer lifecycle hooks around:

- module initialization
- application bootstrap
- application shutdown

That work helps the runtime itself and also improves package integration for features that need a clean bootstrap point.

### Runtime configuration validation

Invalid OpenSwoole configuration now fails early instead of slipping into harder-to-debug runtime startup errors.

## Important status note

OpenSwoole support in `0.8.0` should be treated as **experimental**.

This release establishes the runtime foundations. It does not mean every long-lived runtime feature is complete or production-polished yet.

That is intentional.

We are releasing the runtime at the point where the architecture is real, the worker-safety story is much stronger, and the framework is ready for broader validation.

What `0.8.0` does not claim is just as important:

- WebSockets are not part of this release
- streaming APIs are not part of this release
- this is not yet a blanket promise that every async runtime feature is production-ready

## Documentation

This release is accompanied by:

- OpenSwoole getting-started guidance
- a deeper runtime guide
- an engineering article explaining why the runtime work matters and what comes next

## What comes next

After `0.8.0`, the next major milestone is `0.9.0`, focused on ORM stability.

That work will concentrate on:

- hardening MySQL support
- properly implementing SQLite as a supported target
- bringing PostgreSQL support up to the same level
- making the query builder and SQL generation properly dialect-aware
