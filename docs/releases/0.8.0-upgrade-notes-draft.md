# 0.8.0 Upgrade Notes Draft

## Path

- From: `0.7.6`
- To: `0.8.0`

## Automatic

- `assegai update` can upgrade the relevant Assegai packages for the workspace
- `assegai update` now also replays installed package installers after the version bump
- that means package-driven integrations such as ORM and Events can be repaired automatically where possible

## Manual

- if your project bootstrap still depends on the current working directory, switch it to `AssegaiFactory::createFromProject(AppModule::class, __DIR__)`
- if you want to try OpenSwoole, install the OpenSwoole extension first and then opt in through `assegai.json` or `assegai serve --runtime=openswoole`
- treat OpenSwoole as experimental in `0.8.0`; the default PHP runtime remains the normal path
- if your app uses docs or API export heavily, recheck `/docs`, OpenAPI export, and Postman export after the upgrade

## Verification

- run `assegai update`
- run the normal development server path first with `assegai serve`
- if you are evaluating OpenSwoole, run `assegai serve --runtime=openswoole`
- rerun your application test suite
- smoke test the features most likely to reflect runtime or workspace-root changes:
  - `/docs`
  - OpenAPI export
  - Postman export
  - any package-driven module integration such as ORM or Events
