# 0.8.2 Upgrade Notes

## Path

- From: `0.8.1`
- To: `0.8.2`

## Automatic

- `assegai update` can upgrade the relevant Assegai packages for the workspace

## Manual

### Web component props helper rename

If your templates or PHP code call `web_component_props(...)`, rename those calls to `wc_props(...)`.

If your templates reference `ctx.webComponentProps`, rename those references to `ctx.wcProps`.

No other code changes are required for a normal upgrade from `0.8.1`.

### Public asset serving behavior

If your application deliberately serves files with non-standard extensions from the `public/` directory, verify those extensions are in the permitted allowlist after upgrading. Requests for unlisted extensions will no longer be served.

`.phps` files and hidden files outside `.well-known/` will now be blocked by the framework regardless of what is present on disk.

## Verification

- run `assegai update`
- run `assegai serve` and confirm normal request and static asset handling
- smoke test any web component pages that used the old helper name
- if your app served unusual file types from `public/`, verify those requests still work as expected
- rerun your application test suite
