# 0.8.0 Release Prep

This is the working checklist for the `0.8.0` milestone.

Theme: OpenSwoole Runtime Foundations.

The goal is not to declare the runtime complete. The goal is to make it safe, coherent, and honest enough to ship as an experimental runtime target.

## Done

- runtime abstraction exists in `core`
- OpenSwoole runtime exists in `core`
- CLI runtime selection exists through `serve --runtime=openswoole`
- project config supports OpenSwoole runtime settings
- request-scoped runtime state is isolated across worker requests
- lifecycle hooks exist for module init, application bootstrap, and application shutdown
- shutdown hooks now fire consistently for non-OpenSwoole runtimes too
- runtime failure handling routes back through framework error handling
- runtime tests cover:
  - worker lifecycle
  - sequential request isolation
  - recovery after a failed request in the same worker
  - invalid OpenSwoole config rejection
- OpenSwoole guides are written
- OpenSwoole engineering blog article is drafted

## Still required before tagging 0.8.0

- prepare final release notes
- create the release branch `release/0.8.0`
- run the focused milestone test set on the release branch
- do one manual CLI smoke pass for:
  - `assegai serve --runtime=openswoole`
  - invalid OpenSwoole config failure
  - normal PHP runtime still working

## Nice to have but not required for 0.8.0

- native integration tests against a real installed OpenSwoole environment
- extra runtime metrics or tracing hooks
- richer worker/task-worker management surfaces
- WebSockets
- streaming APIs
- a formal microservices transport

## Release wording guidance

Recommended language:

- say OpenSwoole support is **experimental**
- say `0.8.0` ships runtime foundations
- do not imply WebSockets, streaming, or full async infrastructure are already shipped
- do not market `0.8.0` as a performance release first; market it as a runtime architecture milestone
